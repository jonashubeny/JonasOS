# JonasOS: my DIY Linux "distro" side project

Repository: https://github.com/jonashubeny/JonasOS

## Why I'm doing this
You probably expecting some cool explanation for this, but no. I'm doing this just for fun and for the love of the game. :D 

I just saw a video on YouTube about some guy making his own lightweight distro based on Ubuntu and I found it cool. So I'm starting my own personalized Linux distro with completely no experience.


JonasOS is not meant to become a mainstream Linux distribution. It’s a personal, opinionated developer environment and a public learning journey.

## What will it be
That's really good question! I don't know...

I have a vision about my personalized distro that isn't just another Arch Linux setup. Here are some things I would like to have in my OS:

- Debian based
- Xfce (because I like the look of Kali Linux)
- My own icons, logo, sounds, etc

## What JonasOS won't be
Just to be clear (mostly to future me):

- This is **not** a serious competitor to Debian/Ubuntu/arch.
- This is **not** me trying to invent a new kernel or package manager.
- This is **not** a "perfect" distro with huge user base.
- This in **not** a pentesting distro like Kali (I just like to vibe)

## My first tiny goal (MVP, but I won't pretend it's fancy)
Step 1: I want JonasOS to **boot**

That's it.

If I can boot an ISO in a VM and get:

- a desktop
- internet
- a terminal
- and a couple of tools I actually use

...then the project is already in success.

Everything else is DLC.

## Why Debian-based?
Because:

- Debian feels like a solid base for learning without chaos
- It's stable and boring (in a good way)
- If I break something, it's usually **my fault** (I can't constantly complain)
- It isn't Arch

## The vibe / design goals
I want JonasOS to feel:

- minimal, but not empty
- fast
- smooth
- dev-focused

## How I'll work on this
I really don't want to turn this project into a second job, so I'll set some healthy rules:

- If it stops being fun, I pause it
- Small steps only
- I'll keep a simple changelog and post progress here sometimes

## What I'm expecting to learn
I have no experience with building an ISO, so I'm expecting things like:

- how a live ISO is structured
- where default desktop configs live
- how to bake configs into an image
- how to test changes quickly in VM
- how to not brick my system (hopefully)

## Next steps
My next steps are probably:
1. create the GitHub repo
2. make a basic project structure
3. build the first ugly ISO that boots
4. iterate from there

No timeline. No promises. Just vibes.
